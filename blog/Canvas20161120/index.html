Canvas

发表于 2016-11-20

Canvas简介：

canvas 是 HTML5 提供的一个用于展示绘图效果的标签. canvas 原意画布, 帆布. 在 HTML 页面中用于展示绘图效果.

Canvas 基本的绘图方法：

getContext( ) 方法

语法：Canvas.getContext( typeStr )；
描述：

1. 该方法用于绘制上下文工具
2. 方法可返回一个对象，该对象提供了用于在画布上绘图的方法和属性。
3. 如果是绘制平面图形使用 '2d' 作为参数, 如果绘制立体图形使用 'webgl'.

moveTo( ) 方法：

语法：context.moveTo(x,y);
描述：
1.该方法用于设置绘制起点。
2.参数x，y用于绘制表示在画布中的坐标位置

lineTo( ) 方法：

语法：context.lineTo(x,y);
描述：

1. lineTo() 方法添加一个新点，然后创建从该点到画布中最后指定点的线条（该方法并不会创建线条）。
2. 其中参数 x, y 表示在坐标系中的位置, 分别是 x 坐标与 y 坐标.

stroke( ) 方法：

语法：context.stroke();
描述：

1. stroke() 方法会实际地绘制出通过 moveTo() 和 lineTo() 方法定义的路径。默认颜色是黑色。

总结：

  1. 绘图先要获得上下文, 即绘图工具
  2. 绘图需要设置开始的坐标
  3. 绘图是先描点, 然后一个一个依次连线
  4. 依次绘图只能绘制单一样式( 色彩等 )
  • 如果要绘制图形，需要通过DOM对象得到一个绘制对象。
  • 必须通过canvas的属性来设置画布的大小，不能通过css设置，css会拉伸画布，画布默认宽高为 300 * 150
  • 路径：路径就是对将来绘制图形轮廓的规划，并不是真正的绘制。

路径：

closePath() 闭合路径

语法：context.closePath();
描述：从当前路径的结束点，到起始点相连。

beginPath() 开启新的绘制

语法：context.beginPath();
描述：重新开始一条路径，或重置当前路径；为了防止重绘之前的路径，先把他们清除掉

非零环绕原型：

说明：

  1. 在 canvas 中使用各种方法描点实际上描述的是一个称为路径( path )的东西.
  2. 在 canvas 绘图中, 所有描述的东西都是路径, 只有最后填充或描边的时候才会显示出效果.
  3. 每一个路径都是一个状态

作用：

  • 作用就是用来判断画布中的哪些地方需要填充。
  • 真正作用是用判断哪些局域是被环绕起来

原理：

  • 随便找一块封闭的区域，然后在区域随便找一点，向外发一条射线，
  • 然后会继续计数判断，初始值为0，
  • 如果射线遇到边，这条边相当于点是逆时针，那么-1，顺时针+1，

  • 最终结果非0，那么就认为这块区域是被环绕起来的。

  • 凡是基数边，围起来的图形区域，都是被环绕起来的，如果调用fill方法，一定会被填充

线性相关属性

setLineDash() 设置虚线

语法：context.setLineDash( [ 实线部分的长度，空白部分的长度 ] )
描述：如果传入数组的数量为基数，那么保存时canvas会自动变为偶数的数量存储。

strokeStyle 设置描边色：

语法：context.strokeStyle=’ color ‘ ;
描述：strokeStyle 可以设置描边颜色, 与 CSS 的语法一样

fillStyle 设置填充色：

语法：context.fillStyle= ‘ color ‘ ;
描述：设置用于填充绘画的颜色、渐变或模式。

lineCap 线帽样式

语法：context.lineCap = ‘butt’ || ‘round’ ||’square’
描述：属性设置线条末端线帽的样式

1. 默认值为butt,表示两端使用方形结束。
2. round 表示两端使用圆角结束。
3. square 表示突出的圆角结束。

例：

lineJoin 交点样式

语法：context.lineJoin= ‘bevel’ || ‘round’ || ‘miter’;
描述：设置两条直线的拐点描述方式

1. miter 使用尖角连接，为默认值。
2. round 使用圆角连接。
3. bevel 使用平切连接。

例：

lineWidth 线条宽度

语法：context.lineWidth=number;
描述：设置当前线条的宽度。

矩形方法

rect() 创建矩形

语法：context.rect(x,y,width,height);
说明：x,y为矩形左上角的坐标，width,height 为矩形的宽度和高度

  • 注意：rect 方法就是矩形路径, 还需要使用 fill 或 stroke 才可以看到效果. 因此一般使用 strokeRect 或 fillRect 直接可以看到结果.

fillRect() 绘制填充矩形

语法：context.fillRect(x,y,width,height);
说明：x,y为矩形左上角的坐标，width,height 为矩形的宽度和高度
描述：

1. 用来绘制一个矩形. 比起直接使用 moveTo 和 lineTo 方法要简单许多.
2. 使用该方法不需要使用 moveTo 方法设置起始点, 也不需要调用 stroke 等绘画方法.
3.默认的填充颜色为黑色，可以使用fillStyle 设置颜色样式改变填充颜色。

strokeRect() 绘制矩形(无填充)

语法：context.strokeRect(x,y,width,height);
说明：前两个参数表示绘制矩形的左上角的坐标. 后两个参数表示这个矩形的宽高.

  • 绘制的矩形支持 strokeStyle 设置边框颜色样式.

clearRect() 清除画布

语法：context.clearRect(x,y,width,height);
说明：

1. 用于清除画布中的矩形区域的内容.
2. 参数 x, y 表示矩形区域左上角的坐标, width 与 height 表示擦除矩形区域的宽高.

绘制圆弧

绘制圆弧：

语法：context.arc(x,y,r,sAngle,eAngle,counterclockwise);
描述：

1. 该方法用于绘制一段弧, 配合开始点的位置 与 stroke 方法或 fill 方法可以绘制扇形.
2. 方法中的前两个参数 x, y 表示绘制圆弧的圆心坐标
3. 参数 radius 表示圆弧半径, 单位为弧度
4. 参数 startAngle 与 endAngle 表示开始到结束的角度. 角度以水平向右为 0 弧度, 顺时针为正方向.
5. 参数 anticlockwise 表示是否采用默认的正向角度, 如果传入 true 表示逆指针为正. 该参数可选.

注意：

  1. 使用 arc 绘图的时候, 如果没有设置 moveTo 那么会从开始的绘弧的地方作为起始点. 如果设置了 moveTo, 那么会连线该点与圆弧的起点.
  2. 如果使用 stroke 方法, 那么会从开始连线到圆弧的起始位置. 如果是 fill 方法, 会自动闭合路径填充.

画扇型：

  1. 先设置路径的起点为圆心
  2. 然后通过arc()画弧
  3. 通过closePath方法自动把路径结束点与起点相连

绘制饼图

绘制饼形图最大的特点是角度是叠加的. 开始从 -Math.PI/2 开始绘制, 达到执行角 x 后, 下一个区域从 x 开始绘制, 然后有到一个角 y 停下来. 如此反复到 Math.PI * 3 / 2 结束.

1.三等分饼图

2.根据数据定义角度

根据数据源定义角度, 就是将所有的数据求和, 按照总和为 2 * Math.PI 的结论计算出每一个数据部分的弧度值. 同时颜色可以提前定义好.

绘制文本

绘制文字

语法：context.strokeText( text, x, y, maxWidth )；
context.fillText( text, x, y, maxWidth )；
描述：
1、这两个方法的功能都是在给定的 x, y 坐标下绘制文本内容.
2、stroke 绘制描边文字, 文字内空心. fill 绘制填充文字, 即实心文字.
3、最后一个参数可选, 用于限制文字的总宽. 特殊条件下, 系统会自动调整文字宽度与大小以适应该参数限制.

####设置文字属性

语法：context.font = value ;
描述：
1、该属性用于设置绘制字体的各种信息, 与 CSS 语法一致, 设置字体形状, 样式, 字号粗细等.
2、其顺序可以是: style | variant | weight | size/line-height | family.
3、默认值为 10px sans-serif

####设置字体水平对齐方式

语法：context.textAlign=value;
描述：
1、该属性用于设置文字的水平对齐方式. 设置文字居中, 靠左右对齐等.
2、该属性可以设置的值有: start( 默认 ), end, left, right, center.

####设置字体垂直对齐方式

语法：context..textBaseline = value ；
描述：
1、该方法设置文字在垂直方向上的对齐方式.
2、该属性可以取值: top, middle, bottom, hanging, alphabetic, ideographic
3、基于参考点的直线, 其中 top, middle, buttom 分别表示靠上, 居中, 靠下对齐.
4、alphabetic 表示字母基线, 类似于英文字母的对齐方式. 例如 a, g, f 等字母.
5、ideographic 表意对齐. 使用字母对齐中超出的字母为参考. 即比字母基线略靠下.
6、所有的对齐方式是根据文字特点相关的. 对于中文主要使用的还是 top, bottom 和 middle.

绘制图片

绘制简单图像

语法：context.drawImage(img,x,y);
描述：
1、使用三个参数, 允许在画布上的 任意位置 绘制图片.
2、参数 img 是指图片对象. 可以是 img 标签, 或者是 video 标签, 已经另一个 canvas 等.
3、需要注意的是如果直接添加 img 对象是不可以的, 需要等待其加载.

####在指定区域内绘制图像

当图片比较大的时候, 如果使用这样的方式绘图, 那么图片可能会完全覆盖画布. 因此, 需要将其控制在一个矩形区域内绘制.
语法：context.drawImage(img,x,y,width,height);
描述：
1、参数 Width, Height 表示绘制的矩形区域. 绘制的 dx, dy 坐标决定了开始.
2、该方法绘制的图像会在指定范围内被压缩显示.

绘制任意图像

语法：context.drawImage( 图像资源，裁剪的x坐标，裁剪的y坐标，裁剪多宽，裁剪多高，x坐标，y坐标, 图像显示的宽，图像显示的高 )

制作动画

变换

旋转变换

语法：context.rotate(angle);
描述：
1、该方法将坐标轴进行旋转变换.
2、参数是弧度, 表示旋转的方式. 正数表示顺时针旋转, 负数表示逆时针旋转.

  • 该变换常常用于实现旋转动画等.

平移变换

语法： context.translate(x,y);
描述：
1、该方法将坐标进行平移.
2、x 表示水平移动, 正数向右, 负数向左 ；y 表示垂直移动, 正数向下, 负数向上.
3、变换可以重复调用, 变换是可以累加的.

  • 注意: 变换也会被 “继承”, 开启新路径后依旧保留原先变换.
  • 使用该变换, 常常用于绘制不同位置, 但是形状重复的图形.

缩放变换

语法：context.scale(scalewidth,scaleheight);
描述：
1、该方法实现水平与垂直的缩放.
2、参数 x 控制水平缩放倍率. 传参 1 表示不作缩放, 传入大于 1 的数字表示扩大.
3、参数 y 控制垂直缩放倍率. 传参 1 表示不作缩放, 传入大于 1 的数字表示扩大.

  • 该变换常常用于放大与缩小, 以及反转的效果.

状态：

状态就是变量或者属性不同时期的值。

  1. 状态的保存：就是把绘图对象，所有属性的状态值copy保留一份。
    语法：ctx.save()；
    注意：没有和路径相关的状态，即save不会保存路径。即状态的保存与回滚，与路径没有任何瓜葛。

  2. 状态的回滚：把之前保存的状态拿出来，覆盖当前的状态。
    语法：ctx.resotr();
